javascript-面向对象

面向对象（OOP）​

  1. 程序是干嘛的？
    • 程序就是对现实世界的抽象（照片就是对人的抽象）
  2. 对象是干嘛的？
    • 一个事物抽象到程序中后就变成了对象
    • 在程序世界中，一切皆对象
  3. 面向对象编程
    • 面向对象的编程指，程序中所有的操作符都是通过对象来完成的
    • 做任何是之前都需要先找到它的对象，然后通过对象来完成各种操作

类​

  • 使用Object创建对象的问题：

    1. 无法区分不同类型的对象
    2. 不方便批量创建对象

  • 在JS中可以通过类（class）来解决这个问题：

    1. 类是对象模板，可以将对象中的属性和方法直接定义在类中
      • 定义后可以直接通过类来创造对象
    2. 通过同一个类创建的对象，我们称为同类对象
      • 可以使用instanceof来检查一个实例是否由某个类创建
      • 如果某个对象是由某个类所创建，则我们称该对象是这个对象的实例

  • 语法：

    • class 类名 {} //类名要使用大驼峰命名法
    • const 类名 = class {}

  • 通过类创建对象

    • new 类（）

封装​

面向对象的特点：

  • 封装 多态 继承

  1. 封装

    • 对象就是一个来存储不用属性的容器

    • 对象不仅是存储属性，还要负责数据安全

    • 直接添加到对象中的属性，并不安全，因为它们可以被任意修改

    • 如何确保数据安全

      1. 数据私有化（#）
        • 将需要保护的数据设置为私有，只能在类中操作
      2. 提供setter和getter方法来开放对数据的操作
        • 属性设置私有，通过getter setter 方法操作属性带来好处
          1. 可以控制属性的读取属性
          2. 可以在方法中对属性的值进行验证

    • 封装主要是用来保护数据的安全

      • 实现封装的方式：
        1. 属性私有化 在属性前加#
        2. 通过getter和setter方法来操作属性 
get 属性名(){
    return this.#属性名
}

set 属性名(参数){
    this.#属性名 = 参数
}

class Person {
    #name
    #age
    
    constructor(name, age) {
        this.#name = name
        this.#age = age
    }
    
    sayHello() {
        console.log(this.#name)
    }

    get name(){
        return this.#name
    }

    set name(name){
        this.#name = name
    }

    get age(){
        return this.#age
    }

    set age(age){
        this.#age = age
    }
}

继承​

继承
    - 可以通过extends关键来完成继承
    - 当一个类继承另一个类时，就相当于将另一个类中的所有代码复制到了当前类中（简单理解）
    - 继承发生时，被继承的类称为父类（超类），继承的类被称为子类
    - 通过继承可以减少重复的代码，并且可以在不修改一个类的前提对其进行扩展
    - 通过继承可以在不修改一个类的情况下对其进行扩展
    - OCP 开闭原则
        - 程序对修改关闭 对扩展开放

    封装 —— 安全性
    多态 —— 灵活性
    继承 —— 扩展性

class Animal {
    constructor(name){
        this.name = name
    }

    sayHello(){
        console.log("动物在叫~")

}

class Dog extends Animal{

    sayHello(){
        // 调用父类的sayHello()函数
        super.sayHello() // 在方法中可以使用super来引用父类的方法
        console.log("汪汪汪！")

}

对象中的结构、​

对象中存储属性的区域实际上有两个：
1. 对象自身
    - 直接通过对象所添加的属性，位于对象自身
    - 在类中通过 x = y 形式添加的属性，位于对象自身 
2. 原型对象（prototype）
    - 对象中还有一些内容，会存储到其他的对象里（原型对象）
    - 在对象中会有一个属性用来存储原型对象，这个属性叫做__proto__ （特殊字符开头的基本都是不想让我们访问的）
    - 原型对象也负责为对象存储属性，
        - 当我们访问对象中的属性时，会优先在对象自身查找访问
        - 当对象自身中不存在该属性时，才会去原型中寻找
    - 会添加到原型中的情况：
        1. 在类中使用 xxx(){} 的方式添加方法时 就会存储到原型中
        2. 主动向原型中添加属性和方法

原型对象​

  • 访问一个对象的原型对象

    • 对象.proto
    • Object.getPrototyperOf(对象)
    • 类名.prototype

  • 原型对象中的数据：

    1. 对象中的数据（属性、方法等）
    2. constructor（对象的构造函数）

  • 注意：

    • 原型对象也有原型，这样就构成了一条原型链，根据对象的复杂程度不同，原型链的长度也不同
      • xx实例的原型链: 实例对象 --> 原型 --> Object --> null
      • obj的原型对象：obj对象 --> Object --> null

  • 原型链：

    • 读取对象属性时，会优先在对象自身查找
      • 如果对象中有，则使用，没有则去对象的原型中寻找
      • 如果原型中有，则使用，没有则去原型的原型中寻找
      • 直到找到Object对象的原型（Object的原型没有原型（为null））
      • 如果依然没有找到，则返回undefined

  • 作用域链，是找变量的链，找不到会报错

  • 原型链，是找属性的链，找不到会返回undefined

  • 所有的同类型对象它们的原型对象都是同一个，也就意味着，同类对象的原型链相同

  • 原型的作用：

    • 原型就相当于是一个公共的区域，可以被所有实例访问，
      • 可以将该类实例中，所有公共的属性（方法）统一存储到原型中
      • 这就我们只需要创建一个属性，即可被所有实例访问
    • JS中的继承就是通过原型来实现的
      • 当继承时，子类的原型就是一个父类实例

  • 在对象中有些值是对对象独有的，像属性（name, age, gender）每个对象都有独立的值。

    • 但是有些值对于每个对象来说都是一样的，像各种方法，对于相同的值没必要重复的创建。

修改原型​

  • 大部分情况下，我们是不需要修改原型对象
    • 注意：
      • 千万不要通过类的实例去修改原型
        1. 通过一个对象影响所有同类对象，这么做不合适
        2. 修改原型先得创建实例，麻烦
        3. 危险
  • 处理通过proto能访问对象的原型外，
    • 还可以通过类的prototype属性，来访问实例的原型
    • 修改原型时，最好通过类去修改
    • 好处：
      1. 修改原型就是修改所有实例
      2. 无需创建实例即可完成对实例原型的修改
    • 原则：
      1. 原型尽量不要手动改
      2. 要改也不要通过实例对象去改
      3. 通过 类.prototype 属性去修改
      4. 最好不要直接给prototype赋值
通过对象修改原型，向原型中添加方法，修改后所有同类实例都能访问该方法 不要这么做
p.__proto__.run = () => {
    console.log('我在跑~')
}
p.__proto__ = new Dog() // 直接为对象原型赋值了一个型的原型 不要这么做

new运算符​

new​

  • new运算符是创造对象时要使用的运算符
    • 使用new时，到底发生了哪些事情：
      1. 创建一个普通的JS对象（Object对象 {}）, 为了方便，称其为新对象
      2. 将构造函数的prototype属性设置为新对象的原型
      3. 使用实参来执行构造函数，并且将新对象设置为函数中的this
      4. 如果构造函数返回的是一个非原始值，则该值会作为new运算的返回值返回（千万不要这么做） 如果构造函数的返回值是一个原始值或者没有指定返回值，则新的对象将会作为返回值返回 通常不会为构造函数指定返回值
